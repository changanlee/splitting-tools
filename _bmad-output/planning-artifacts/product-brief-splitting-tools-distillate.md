# splitting_tools — PRD 輸入精煉包

> 上游：brainstorming SSOT（18 ideas，已完成）+ 競品研究 + 3 道 review。本檔為非執行摘要的全細節，供 create-prd 直接消化。

## 產品本質
- 個人/朋友用「Costco 收據拍照分帳」小工具。**明確非商業產品**，無帳本、無金流、無商業模式。
- 痛點：(B) 回家對天書收據沒人想算（動機問題）；(D) 整箱誰拿幾個吵不清（時機問題）。
- 一句話：付款人拍收據 → 視覺 LLM 解析+對帳 → 付款人核對 → 出免註冊連結 → 朋友自助認領（未認領＝PENDING）→ 單品項可加權多人分 → 每人應付。

## 核心流程（v1，7+ 步，視為需求 hint）
1. 拍照（前端壓縮，長邊~1600px）→ 單次視覺 LLM 呼叫；job_id + 輪詢（>1s 不阻塞）。
2. LLM 還原縮寫品名、配對 IRC 即時折扣到母品項。
3. 發起人核對關卡＝**總額對帳 + 可疑行肉眼抽檢（單行佔比過高/品名互換）+ 每筆 IRC→母品項一鍵改綁**；可編輯金額/品名/折扣歸屬/增刪行。
4. 閘門有底有逃生口：總計糊掉可手動輸入；連續 N 次兜不攏可「未驗證強制放行」+ 對所有認領者顯示未驗證橫幅。**永不卡死付款人**。
5. 產生不可猜連結 + 可貼訊息卡（日期/總額/品項數/付款人，非裸 URL，防像詐騙）。
6. 朋友點開：**身份綁本機 token**（非名字字串）；同裝置再開＝同一人；進入可從既有名單挑「是不是你」；每人只能改自己裝置的認領；可按「我認領完了」。
7. 未認領＝PENDING；付款人看即時「N 項/¥Z 未認領」橫幅；**必須主動按「結束分帳並吸收剩餘」才定案**（非超時/靜默）。
8. 結算頁：極簡誰欠誰 + 一鍵複製可貼回群組純文字 + 「✓對得上總計¥X」信任標記給認領者 + 收據縮圖可查 + 付款人剩餘明示為「其餘預設付款人的」+ 同團名字 localStorage 記憶。
- 分攤：單品項多人，預設均分，可改份額 A:5/B:3（Splitwise shares 心智）；拿量比 10:20＝shares 1:2，v1 已可表達整箱比例。
- 稅金：無獨立稅攤（VAT 內含每行）；**v1 硬鎖只接受 #5564 同結構收據，異結構即拒絕**。

## 已拍板決策（locked 2026-05-18）
- 未認領＝PENDING + 付款人顯式吸收（**正式取代**原「未認領靜默歸付款人」）。
- 身份＝本機 token（原 #21 認領完整性之定案解）。
- 稅：無獨立稅攤 + 同結構硬鎖 + 異結構拒絕；比例攤稅 fallback 移 v2 backlog（v1 不做）。
- 資料：上傳前遮蔽會員卡號；結清後 session 凍結唯讀；連結 30 天到期即銷毀影像/claims。
- OCR＝視覺 LLM 單次呼叫，無傳統 OCR stack。POC 固定測資＝使用者真實蘇州 Costco #5564，寫成可執行斷言 `parsed_sum == 2208.50`（含 3 筆 IRC 折扣抵減）。

## 明確 Rejected（勿再提）
- 護城河/差異化/對標 Splitwise — 使用者明令移除（這是創業思維，會把小工具做爛）。
- 長期滾動帳本/關係黏著、金流/收款閉環、地理圍欄推播、語音分帳、遊戲化搶貨、商業模式、跨幣別報帳/發票。
- 「停車場 geofence 推播花俏版」— 只保留「結帳後當下、低摩擦」精神，不做偵測推播。
- 不切實際的「拍照→連結 <1 分鐘」硬指標（含外部 LLM 呼叫不現實）。

## 技術約束（side-project 標準，非協商）
- LLM 邊界：指數退避重試、結構化 log（model/tokens/latency/cost/session_id）、成本追蹤**持久化於 Postgres**（非記憶體）、>1s 走 job_id+輪詢、parse 端點 **per-session/per-IP 預算上限**（免註冊開放連結防濫用/grief）。
- 後端：單一全端服務 + Postgres 存 session/claims；DAU<10k 不引入 microservices/websocket（v1 用輪詢）。
- 安全：不可猜連結 ID、結清凍結、session 到期；認領變更需 undo + 付款人可見變更紀錄。
- n=1 風險：可行性僅單收據驗證，**準確率未知**；對帳只證總額、不證逐行品名/IRC 歸屬 → 故需肉眼抽檢層。v1 須建 3–5 張不同狀態（褪色/過曝/超長/折疊）回歸測資。

## 競品情報（借成熟模式，非對標）
- Splitwise：多模式分攤但逐項僅 0/50/100% + 付費牆 → 借其 shares/units 模型。
- Tab/Splyt：拍收據 tap-to-claim 有先例；Splyt「強迫先邀朋友才能分」被罵爆 → 反面教材，零守門。
- Tricount/Splid：零註冊被讚、結算頁極簡+最少轉帳 → 借免帳號連結與極簡結算。
- WeChat 群收款：可見「誰已/未」狀態＝催人付唯一有效解（不需金流）→ 借狀態看板（v1 輪詢版）。

## Scope 訊號
- v1 IN：上 8 點流程 + 加權份額 + #5564 回歸斷言 + 同團名字記憶。
- v1 OUT / v2 backlog：按數量拆專屬 UI（#13；份額已可替代）、即時看板真同步、結算自動最少轉帳、非 Costco 通用化、比例攤稅 fallback。永久 OUT：金流/帳本/商業化。

## 成功判據（單次真實 dogfood 可觀察清單）
- 全程賣場/車上完成、零事後群組補算；付款人手動修正≤~3 項過關；無「未認領被靜默吃掉」；朋友端無需口頭解釋即完成；總額對帳通過（前置條件非成果）。

## 仍開放（PRD 階段處理）
- 「可疑行」判定閾值與抽檢 UX 細節；IRC 配對演算法策略；未驗證強制放行的下游影響；輪詢頻率/競態（多人同時認領同品項）；name-list「是不是你」的具體 UX。
